# Study Companion Backend

FastAPI + MongoDB backend for the Study Companion app.

## Setup

1. Create a virtual environment and install dependencies:
   ```
   python -m venv venv
   venv\Scripts\activate   (Windows)
   source venv/bin/activate   (Mac/Linux)
   pip install -r requirements.txt
   ```

2. Copy `.env.example` to `.env` and fill in real values:
   - `MONGODB_URI` — from MongoDB Atlas (create a free cluster, get the connection string)
   - `JWT_SECRET_KEY` — any long random string
   - `GEMINI_API_KEY` — from Google AI Studio (free tier)
   - `CORS_ORIGINS` — your frontend's URL(s), comma-separated

3. Run the server:
   ```
   uvicorn app.main:app --reload
   ```
   Visit `http://localhost:8000/docs` for interactive API docs (auto-generated by FastAPI).

## What's real vs. what needs testing

**Structurally complete, should work as-is:**
- Auth (signup/login, JWT issuance and verification)
- SM-2 algorithm (`app/services/sm2.py`) - written from scratch per spec
- All routes/schemas matching the frontend's exact data shapes
- CORS configuration

**Needs real-world testing before you trust it:**
- `app/services/whisper_service.py` — untested against Render's actual free-tier memory limits. Start with the "tiny" Whisper model if "base" is too slow/crashes.
- `app/services/gemini_service.py` — the prompt and JSON parsing are a first attempt; Gemini's actual response format needs verifying against a real API key. Print the raw response the first few times you call it.
- `app/services/apkg_service.py` — uses the `genanki` library to avoid hand-building Anki's SQLite schema. Test the exported file by actually importing it into Anki before trusting the format is fully correct for your use case.
